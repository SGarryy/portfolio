The War Room // how I actually think about data problems
🧠 When I reach for which model
Situation My Go-To Why
Need to explain to stakeholders Logistic Regression Coefficients tell the story directly — no black box
Tabular data, mixed features XGBoost Handles missing values, non-linear patterns, fast to tune
Small dataset, high variance Random Forest Bagging smooths out the noise without overfitting
Finding natural clusters K-Means → DBSCAN K-Means to explore shape, DBSCAN if clusters are weird
Text classification baseline TF-IDF + LR Always start simple — transformers only if baseline fails
Time series forecasting ARIMA → Prophet ARIMA for stationary, Prophet when there's seasonality

⚙️ My problem-solving process
1
Define the Question
What decision does this enable? Who uses it?
2
EDA First, Always
Distributions, nulls, outliers, correlations — before any model
3
Dumb Baseline
Logistic regression or mean prediction — what am I beating?
4
Feature Engineering
Where the real gains live. Model choice matters less
5
Evaluate Honestly
Right metric for the problem — not just accuracy
6
Communicate the Insight
The model is not the output — the decision is
🔥 Hot takes

Feature engineering beats model tuning. Every time. No contest.
Accuracy is a terrible metric for imbalanced data and everyone still uses it.

📉 Bias-Variance tradeoff — my mental model
High bias = underfitting — model too simple, missing patterns
High variance = overfitting — model memorised, won't generalise
The goal is the sweet spot — tuned by cross-validation
